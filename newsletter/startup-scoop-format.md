# OM Startup Scoop Format

This document captures the recurring Opportunity Machine newsletter pattern shown in the current Mailchimp HTML export. Use it as the format guardrail for future issues, including Innovate South recaps, current events, programming updates, ecosystem opportunities, founder resources, alumni wins, partner spotlights, and jobs.

## Production Model

The newsletter is Canva-first and Mailchimp-assembled.

1. Decide the issue purpose and audience.
2. Select the most useful sections for founders this month.
3. Draft each section as a short, standalone module.
4. Create or export the section graphics from Canva.
5. Assemble in Mailchimp using the existing OM header, nav tabs, body blocks, buttons, dividers, and footer.
6. Review links, assets, source state, and approvals before scheduling.

## Current Template Anatomy

- Preheader: browser-view link plus hidden preview text.
- Header: OM logo, contact email, phone number.
- Image nav tabs: About, Membership, Programs, Events, Contact.
- Hero image: usually a Canva-generated top graphic for the issue.
- Repeating content module:
  - Canva image or section banner.
  - Short headline.
  - 1 to 3 concise paragraphs.
  - One primary CTA button or inline link.
  - Divider.
- Mixed layout module:
  - Left/right image and text pairing for resource, partner, alumni, or mentor spotlights.
- Jobs/talent module:
  - Compressed list grouped by company.
  - Links directly to postings.
  - CTA to join the startup jobs list or submit openings.
- Footer:
  - Website/social links.
  - Mailchimp unsubscribe, preference, and address footer.

## Editorial Slots

Every issue does not need every slot. Pick the strongest 4 to 7 modules.

| Slot | Job | Best For | CTA Pattern |
| --- | --- | --- | --- |
| Lead | Set the issue frame | Timely theme, recap, big announcement | Read more / register / apply |
| Program Push | Move founders into OM support | Builder, mentor hours, workshops, cohort deadlines | Apply / schedule / learn more |
| Founder Story | Show founder movement | Alumni win, OM Docs, funding, launch, partnership | Watch / read / celebrate |
| Ecosystem Ask | Activate the network | Reports, surveys, policy input, ecosystem data | Take the survey / contribute |
| Opportunity | Surface time-sensitive value | Competitions, grants, accelerators, funding | Register / apply |
| Resource | Teach a useful founder concept | Books, frameworks, tools, office hours | Explore / borrow / use |
| Mentor/Partner | Turn relationships into support | Mentor profile, partner program, sponsor resource | Book / connect / learn |
| Jobs/Talent | Support hiring and talent flow | Startup jobs, internship opportunities | View jobs / submit job |
| Community Events | Help founders find rooms | Meetups, partner events, demo days | RSVP / see events |
| News Intake | Invite member/alumni updates | Milestones, partnerships, acquisitions | Submit your news |

## Copy Rules

- One module, one job.
- Write for founders first, then partners and ecosystem supporters.
- Keep paragraphs short. Most modules should fit in 80 to 140 words.
- Make the value explicit: why this matters, who it is for, and what to do next.
- Avoid generic hype. Tie the item to founder movement, resource access, relationship capital, opportunity flow, or ecosystem feedback.
- Separate confirmed facts from draft/internal context.
- Do not cite metrics, rankings, funding, awards, deadlines, or eligibility unless the source has been checked.

## Canva Asset Rules

- Use Canva for hero graphics and section banners.
- Export images at least 600px wide for full-width Mailchimp blocks.
- Use concise graphic text, because the email body should carry the details.
- Add alt text for every image in Mailchimp.
- Avoid making critical details image-only. Deadlines, URLs, eligibility, and CTAs should appear in live text too.

## Link Rules

- Every module gets one primary CTA.
- Use UTM links for recap and campaign traffic when practical.
- Verify external links before send.
- For time-sensitive opportunities, include the deadline in live text and verify the date on the source page.
- Use Mailchimp's preview/test flow before scheduling.

## Source States

Label each item before drafting:

- `approved_public`: safe to publish as-is.
- `review_ready`: drafted but needs staff/stakeholder approval.
- `source_needed`: promising item, but source link or fact check is missing.
- `asset_needed`: copy is usable, but Canva graphic or image is missing.
- `permission_needed`: founder, partner, or photo usage needs approval.
- `do_not_publish`: internal, unverified, sensitive, or not aligned.
