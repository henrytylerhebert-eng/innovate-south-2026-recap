# Startup Scoop Review Checklist

Use this checklist before sending or scheduling an OM newsletter in Mailchimp.

## Strategy

- [ ] The issue has one clear purpose.
- [ ] The lead item is the most useful or urgent item for founders.
- [ ] The issue is not just a digest; it shows founder movement, resource activation, opportunity flow, or ecosystem support.
- [ ] The selected modules fit the audience and send timing.

## Source And Approval

- [ ] Each module has a source state.
- [ ] Public claims are confirmed by a source or staff reviewer.
- [ ] Founder, partner, award, funding, and job details are verified.
- [ ] Photos and founder stories have permission where needed.
- [ ] Sensitive/internal items are removed.
- [ ] Any unknown facts are marked `Unknown` instead of guessed.

## Copy

- [ ] Subject line is clear and specific.
- [ ] Preview text adds context instead of repeating the subject.
- [ ] Each module has one job and one primary CTA.
- [ ] Copy is short enough for scanning.
- [ ] Deadlines, locations, and eligibility are in live text, not only graphics.
- [ ] Founder benefit is explicit.
- [ ] Tone feels like OM: practical, founder-first, ecosystem-aware, and optimistic without hype.

## Canva And Assets

- [ ] Hero graphic is uploaded and displays correctly.
- [ ] Section banners are uploaded and display correctly.
- [ ] Every image has alt text.
- [ ] Important details are not trapped only inside images.
- [ ] Image links point to the intended CTA where appropriate.

## Mailchimp Build

- [ ] Header/logo/contact area is correct.
- [ ] Top navigation links are current.
- [ ] Buttons render correctly.
- [ ] Dividers and spacing look consistent.
- [ ] Footer includes social links, preference link, unsubscribe link, and mailing address.
- [ ] No placeholder copy, broken merge tags, or stale campaign names remain.

## Links And Tracking

- [ ] Every button and image link has been clicked in preview.
- [ ] External links open to the expected destination.
- [ ] UTM links are used for campaign-owned pages when practical.
- [ ] Forms, Calendly links, surveys, and registration links work.
- [ ] Time-sensitive links are still open.

## Final QA

- [ ] Previewed in Mailchimp desktop view.
- [ ] Previewed in Mailchimp mobile view.
- [ ] Test email sent to internal reviewers.
- [ ] Test email checked in at least one desktop inbox and one phone inbox.
- [ ] Final approver signed off.
- [ ] Send or schedule time confirmed.
